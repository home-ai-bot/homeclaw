## 🌟 产品定位

`homeclaw` 是专为 AI 时代重构的智能家居中枢。我们摒弃传统配置型系统的复杂逻辑，以 **大模型理解 + 小模型路由 + 本地工作流引擎** 为核心，打造真正零门槛、高隐私、全开放的智能家庭操作系统。

- 🗣️ **自然语言即控制**：无需学习术语、编写规则或调试协议，对话即可完成设备接入、场景编排与权限管理。
- 🧠 **AI 原生架构**：大小模型协同分工，复杂意图实时解析，轻量工作流本地执行，响应快、断网可用。
- 🔒 **隐私与安全优先**：核心推理与设备控制闭环于局域网，支持纯本地大模型部署，数据不出户。
- 🌍 **开放与平权**：设备回归纯粹执行，大脑统一调度。通过标准 Skill 契约无缝接入主流生态，旧硬件焕发新生。

## ✨ 核心特性

| 特性 | 说明 |
|:---|:---|
| 🌐 **极简接入** | 原生支持米家、涂鸦、HomeKit、Matter 等主流生态。提供标准 Skill 契约，设备发现、状态同步、控制指令一键纳管。 |
| 🧠 **智能平权** | 设备只需提供控制接口，思考、调度、联动全由 `homeclaw` 大脑接管。智能门锁、喂食器、扫地机彻底“去脑化”，专注执行。 |
| 💰 **极低部署成本** | 完美适配树莓派、NAS、旁路由、旧手机等闲置硬件。无大模型仅需 50MB 内存，有本地模型 8GB 即可流畅运行。 |
| 🔒 **本地化安全** | 内置本地大模型支持，所有数据、推理、指令均在局域网内完成。支持云端模型按需调用，隐私绝对可控。 |
| 🚀 **无限场景扩展** | 从监督写作业、防火防盗，到自动喂食、智能烹饪。支持空间感知控制、习惯学习与动态设备添加，场景无边界。 |

### ⚡ 操作

#### 🔌 Client 接口实现

```go
// 获取家庭列表
GetHomes()

// 获取房间列表
GetRooms()

// 获取设备列表
GetDevices()

// 单个设备操作
GetSpec()    // 获取设备规格
GetProps()   // 获取设备属性
SetProps()   // 设置设备属性
Execute()    // 执行设备动作

// 事件管理
event.Enable()   // 启用事件订阅
event.Disable()  // 禁用事件订阅
```

**本地控制特性**：支持本地控制，可接受设备事件推送。

#### 🛠️ Skill 方式

```go
// 获取家庭列表

```

## 🎯 典型场景

### 🏠 空间感知控制

- **智能识别**：说"开灯"自动识别当前房间/成员常驻位置，无需每次指定
- **全局操作**：支持"关掉所有灯"等全屋控制指令
- **精准控制**：支持"打开客厅的灯"等指定设备控制

### 🤖 动态设备添加

- **增量扫描**：新设备入网后，通过对话或音箱语音即可触发增量扫描
- **AI 推断**：AI 自动推断设备归属（房间、用途）

### 📚 习惯学习与个性化

- **偏好记录**：记录用户偏好（如睡眠场景保留台灯）
- **自动创建**：自动创建个性化工作流变体
- **智能适配**：下次触发相同意图时智能适配个人习惯

### 👨👩👧 多用户权限隔离

- **权限分配**：管理员分配房间/设备权限给不同成员
- **身份绑定**：支持通过 IM 渠道绑定身份
