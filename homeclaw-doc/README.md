## 🌟 产品定位

`homeclaw` 是专为 AI 时代重构的智能家居中枢。我们摒弃传统配置型系统的复杂逻辑，以 **大模型理解 + 小模型路由 + 本地工作流引擎** 为核心，打造真正零门槛、高隐私、全开放的智能家庭操作系统。

- 🗣️ **自然语言即控制**：无需学习术语、编写规则或调试协议，对话即可完成设备接入、场景编排与权限管理。
- 🧠 **AI 原生架构**：大小模型协同分工，复杂意图实时解析，轻量工作流本地执行，响应快、断网可用。
- 🔒 **隐私与安全优先**：核心推理与设备控制闭环于局域网，支持纯本地大模型部署，数据不出户。
- 🌍 **开放与平权**：设备回归纯粹执行，大脑统一调度。通过标准 Skill 契约无缝接入主流生态，旧硬件焕发新生。

## 🏗️ 架构设计

`homeclaw` 采用分层解耦架构，确保高扩展性与本地优先原则：

```
[交互层] Web UI(向导) / IM渠道(QQ/钉钉/Telegram) / 智能音箱 (小爱/天猫精灵)
    ↓
[AI 决策层] 小模型 (实时意图分类/路由) + 大模型 (复杂工作流生成/配置引导)
    ↓
[工作流引擎] 轻量 JSON/YAML 定义，支持对话触发/设备事件/定时任务，本地直接执行
    ↓
[设备接入层] Skill 插件化架构，统一发现/读写/订阅/元数据契约
    ↓
[硬件节点层] 树莓派 / NAS / 旁路由 / 超轻量板卡（全平台兼容）
```


### 🤖 AI 分工策略
| 角色 | 触发时机 | 职责 |
|:---|:---|:---|
| **大模型** | 首次配置、生成自动化、复杂问题 | 理解意图、生成工作流定义、设备归属推断 |
| **小模型** | 每次用户交互、设备事件 | 实时意图分类、工作流路由匹配 |
| **工作流引擎** | 小模型路由后 | 本地直接执行设备指令，不再依赖 AI |

## 🔌 设备接入与 Skill 契约

各生态设备通过 `Skill` 插件化接入，无需厂商自行开发复杂逻辑，只需遵循统一接口契约：

| 接口 | 说明 |
|:---|:---|
| `发现` | 扫描局域网设备并注册到 HomeClaw 设备表 |
| `读状态` | 查询设备当前状态（温度/亮度/开关等） |
| `写控制` | 执行控制指令（开灯/调温/锁门等） |
| `订阅变化` | 设备状态变更时主动推送（事件驱动） |
| `元数据` | 描述设备能力与参数（供 AI 理解与调用） |

**本地控制支持情况**：
- ✅ **米家**：miot-spec 支持局域网直连（miio 协议）
- ✅ **HomeKit**：HAP 协议天然本地优先
- ✅ **Matter**：标准设计本地优先，支持事件订阅
- ⚠️ **涂鸦**：部分设备支持 Local API + WebSocket，部分强依赖云端

## 🚀 快速开始

### 📦 部署方式
支持直接运行二进制、源码编译或 App 直接安装。推荐在 Linux/ARM 环境下运行：
```bash
# 1. 下载对应平台二进制包
wget https://github.com/yourname/homeclaw/releases/download/v0.1.0-alpha/homeclaw_linux_amd64.tar.gz
tar -zxvf homeclaw_linux_amd64.tar.gz

# 2. 启动服务（默认读取 ./config.json）
./homeclaw start
```

### 📋 配置说明

- **配置文件格式**：JSON，支持热更新，无需重启服务
- **首次启动**：将自动进入 Web UI 向导，引导完成 AI 模型配置、渠道绑定与设备扫描
- **环境变量**：支持环境变量覆盖核心参数，便于容器化部署

### 💻 硬件要求

| 模式 | 内存 | 硬盘 | 适用场景 |
|:---|:---|:---|:---|
| 纯本地控制 | ≥ 50 MB | ≥ 100 MB | 基础设备联动、IM/音箱控制 |
| 本地大模型 | ≥ 8 GB | ≥ 20 GB | 完整 AI 决策、复杂场景生成、隐私敏感环境 |

## 🎯 典型场景

### 🏠 空间感知控制

- **智能识别**：说"开灯"自动识别当前房间/成员常驻位置，无需每次指定
- **全局操作**：支持"关掉所有灯"等全屋控制指令
- **精准控制**：支持"打开客厅的灯"等指定设备控制

### 🤖 动态设备添加

- **增量扫描**：新设备入网后，通过对话或音箱语音即可触发增量扫描
- **AI 推断**：AI 自动推断设备归属（房间、用途）

### 📚 习惯学习与个性化

- **偏好记录**：记录用户偏好（如睡眠场景保留台灯）
- **自动创建**：自动创建个性化工作流变体
- **智能适配**：下次触发相同意图时智能适配个人习惯

### 👨👩👧 多用户权限隔离

- **权限分配**：管理员分配房间/设备权限给不同成员
- **身份绑定**：支持通过 IM 渠道绑定身份
